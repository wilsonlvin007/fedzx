# 技术文档 04｜部署运维与故障排查（生产服务器）

本篇给出「在阿里云单机」的标准部署与排查清单，适用于你当前的运行方式：Nginx + systemd + Node。

---

## 1. 生产环境组件

- Nginx
  - `fedzx.com`：静态站（`/var/www/fedzx`）
  - `cms.fedzx.com`：反代到 `127.0.0.1:3100`
- systemd
  - 服务：`fedzx-cms`
- Node / npm
  - CMS 运行时
- SQLite
  - DB：`/opt/fedzx/apps/cms/prod.db`

---

## 2. 一键更新上线（常用）

### 2.1 更新 CMS（代码 + 迁移 + 重启）

```bash
cd /opt/fedzx && git pull

cd /opt/fedzx/apps/cms
npm ci

set -a
. /opt/fedzx/apps/cms/.env.production
set +a

npx prisma migrate deploy
npx prisma generate
npm run build

sudo systemctl restart fedzx-cms
sudo systemctl status fedzx-cms --no-pager | head -n 30
```

### 2.2 更新静态站（前台）

```bash
cd /opt/fedzx && git pull

sudo rm -rf /var/www/fedzx/*
sudo cp -a /opt/fedzx/apps/site/. /var/www/fedzx/

sudo nginx -t && sudo systemctl reload nginx
```

## 3. 证书与 HTTPS 排查

### 3.1 检查线上证书（最可靠）

```bash
echo | openssl s_client -connect cms.fedzx.com:443 -servername cms.fedzx.com 2>/dev/null \
  | openssl x509 -noout -subject -issuer -dates -ext subjectAltName
```

期望：

- SAN 包含 `DNS:cms.fedzx.com`

### 3.2 常见错误：ERR_CERT_COMMON_NAME_INVALID

原因：

- Nginx 某个 `listen 443 ssl` 的 server block 挂错证书
- 或存在“抢占默认 443”的冲突 server block

排查：

```bash
sudo nginx -T 2>&1 | egrep -n "server_name cms\\.fedzx\\.com|listen 443|ssl_certificate" | head -n 200
```

## 4. 502 / 500 排查

### 4.1 CMS 502（Nginx 反代失败）

步骤：

```bash
sudo systemctl status fedzx-cms --no-pager | head -n 50
sudo journalctl -u fedzx-cms -n 200 --no-pager
sudo ss -lntp | grep 3100 || true
```

### 4.2 Prisma 报错

常见：

- `P2021 table does not exist`：没跑迁移或数据库指错
- `P3005 schema is not empty`：需要 baseline

baseline（示例）：

```bash
cd /opt/fedzx/apps/cms
set -a
. ./.env.production
set +a

npx prisma migrate resolve --applied 20260315160310_init
npx prisma migrate deploy
```

## 5. Nginx 路由要点

静态站核心：

- `root /var/www/fedzx;`
- `location / { try_files $uri $uri/ /index.html; }`

Docs 必须单独配置（否则 `/docs/<slug>/` 会回落到首页，或出现正文渲染异常）：

```nginx
location ^~ /docs/ {
  try_files $uri $uri/ /docs/_doc/index.html;
}
```

## 6. 备份与恢复（建议）

SQLite 备份（停机可选，推荐在低峰）：

```bash
sudo systemctl stop fedzx-cms
cp -a /opt/fedzx/apps/cms/prod.db "/opt/fedzx/apps/cms/prod.db.bak.$(date +%Y%m%d%H%M%S)"
sudo systemctl start fedzx-cms
```

恢复：

```bash
sudo systemctl stop fedzx-cms
cp -a /opt/fedzx/apps/cms/prod.db.bak.YYYYMMDDHHMMSS /opt/fedzx/apps/cms/prod.db
sudo systemctl start fedzx-cms
```
