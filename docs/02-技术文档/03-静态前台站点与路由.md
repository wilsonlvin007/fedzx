# 技术文档 03｜静态前台站点与路由（apps/site）

本篇解释 `fedzx.com` 前台的页面结构、渲染方式与 Nginx 路由规则。

---

## 1. 前台定位

前台是纯静态站：

- 不引入 React/Vue 等复杂框架
- 页面以 HTML 为骨架
- 数据通过浏览器端 JS 请求 CMS API 再渲染

优势：

- 架构稳定、易部署
- Nginx 直接服务，性能好
- 运营改内容只动数据库，不需要重新构建前台

---

## 2. 前台页面清单

目录：`apps/site/`

- `/index.html`
  - 首页：Hero + 热门问题 + 最新文章 + 分类计数 + 推荐文章
  - 数据来自 `GET /api/public/articles`
- `/articles/index.html`
  - 文章列表页：支持搜索与 `?tag=...` 过滤
- `/article.html?slug=xxx`
  - 文章详情页：按 GEO 顺序展示
- `/docs/index.html`
  - 知识库目录页：分类（左）+ 文档列表（右）
- `/docs/_doc/index.html`
  - 通用文档渲染页：`/docs/<slug>/` 都回落到它渲染

---

## 3. API Base 与跨域

前台所有 API 请求通过 `apps/site/js/site-api.js` 封装：

- 读取 `<meta name="fedzx-cms-base" content="...">`
- 生产：`https://cms.fedzx.com`

这样做的目的是：

- 本地/生产可切换
- 静态站不需要写死域名

---

## 4. 文章渲染规则（GEO）

### 4.1 列表页卡片

优先展示：

1. `shortAnswer` / `short_answer`
2. fallback `summary`

并展示：

- 标题
- 发布时间
- 主要标签（第一个 tag）

### 4.2 详情页结构

顺序：

1. 标题（h1）
2. 核心结论（Short Answer 高亮卡片；旧文章 fallback summary）
3. Summary（可选）
4. Body（正文 Markdown）
5. Sources（可选）
6. 标签与相关推荐

### 4.3 Markdown 渲染能力

文章正文和 Docs 都共用轻量 Markdown 渲染器：

- `apps/site/js/markdown-lite.js`

当前支持：

- 标题（`# / ## / ###`）
- 无序列表 / 有序列表
- 加粗 / 斜体
- 行内代码
- fenced code block
- blockquote
- 表格（GFM 风格）

说明：

- 这里没有引入 React 版 Markdown 组件，是因为前台站点保持纯静态 HTML + JS
- 为了不把前台架构复杂化，我们采用了“自带渲染器 + CSS 样式增强”的方案

---

## 5. Docs（知识库）渲染

Docs 分两块：

1. 目录页 `/docs/`
   - 调用 `GET https://cms.fedzx.com/api/internal/docs`
2. 文档页 `/docs/<slug>/`
   - 调用 `GET https://cms.fedzx.com/api/internal/docs/<slug>`
   - 使用轻量渲染器 `apps/site/js/markdown-lite.js` 将 Markdown 转为 HTML

注意：

- 文档页会“提取 Markdown 首行 # 标题作为页面标题”，并从正文中剥离，避免双标题。

---

## 6. Nginx 路由（关键）

前台站点的 Nginx 核心配置（生产在 `/etc/nginx/conf.d/fedzx-prod.conf` 类似文件里）：

- 静态根目录：`root /var/www/fedzx;`
- SPA fallback：
  - `location / { try_files $uri $uri/ /index.html; }`

Docs 需要单独 location，否则 `/docs/<slug>/` 会被 fallback 到 `/index.html`：

```nginx
location ^~ /docs/ {
  try_files $uri $uri/ /docs/_doc/index.html;
}
```

这个规则的意义是：

- `/docs/` 命中真实文件 `/docs/index.html`
- `/docs/<slug>/` 不存在真实目录时，回落到 `/docs/_doc/index.html` 由 JS 渲染正文

---

## 7. 常见故障定位（前台）

1) `/docs/<slug>/` 一直加载中

- 先检查 Nginx 是否回落到 `_doc/index.html`
  - `curl -I https://fedzx.com/docs/<slug>/`
  - 期望 `content-type: text/html`
- 再检查 API 是否能取到文档：
  - `curl -i https://cms.fedzx.com/api/internal/docs/<slug>`

2) 首页/列表没有文章

- 检查 API：
  - `curl https://cms.fedzx.com/api/public/articles`
- 检查文章是否 PUBLISHED
- 检查静态站的 `meta fedzx-cms-base` 是否正确
